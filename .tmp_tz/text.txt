РОЛИ И ПРАВА ДОСТУПА5.1 ПреподавательВидит свои занятия на текущий день.Может открыть только текущее занятие (status = active).Не может открыть занятие повторно после его завершения.Не может вручную менять статус ClassSession.Может выставлять любые статусы посещаемости до окончания занятия.Не может сохранить журнал при наличии хотя бы одного NULL-статуса.Если преподаватель открыл занятие и без нажатия кнопки Сохранить закрыл вкладку, т.е занятие finished и существуют NULL → они автоматически заменяются на П.После окончания занятия может только просматривать его.Видит свои отчёты.Имеет доступ к персональным данным студентов только в рамках закрепленных за ним групп.Может просматривать историю посещений по своим дисциплинам за весь текущий семестр.5.2 КураторМожет подтверждать или отклонять Б_pending.Может делать это только после окончания занятия по расписанию.Может выставлять статус А.Может заменить НБ на А.Может изменять НБ → А только после завершения занятия, причем только если server_time > end_time, независимо от текущего значения status.Может устанавливать А до конца семестра.Видит отчёты по своим группам.Получает уведомления или видит отдельный список (dashboard) со всеми активными запросами Б_pending по своим группам.Статус А может выставлять только куратор.5.3 Учебная частьМожет отменить занятие (cancelled).Устанавливает даты начала и окончания семестра.Имеет расширенный поиск студентов.Видит аналитические отчёты.5.4 АдминистраторИмеет доступ к логам.Не может изменять посещаемость.Не может отменять занятия.5.5 РуководствоДоступ ко всем отчётам без ограничений.6. ОПИСАНИЕ ИНТЕРФЕЙСА16.1. Экран входа (Lgin):Элементы: Поля «Логин» и «Пароль», кнопка «Войти».Логика: При ошибке авторизации – лаконичное сообщение «Неверный логин или пароль». После успешного входа – редирект на Dashboard согласно роли.16.2. Экраны Преподавателя:Dashboard: Список карточек занятий на сегодня. Каждая карточка: время, дисциплина, группа, статус (scheduled/active/finished). Активна для нажатия только та карточка, занятие по которой идёт, по времени, прямо сейчас. Кнопка Просмотр отчётовЖурнал: Таблица студентов с кнопками-переключателями [ П ] [ О ] [ НБ ] [ Б ]. П зелёного цвета, О – жёлтого, НБ – красного, Б – синего. Сверху должны быть надписи с названиями Предмета и Группы. Кнопки «Сохранить» и «Выход» под списком студентов. В процессе сохранения кнопка “Сохранить” меняется на серую “Сохранение...”Экран «Отчеты Преподавателя»:Фильтры: Выбор группы, дисциплины и периода (даты «с» и «по»).Таблица: Список студентов, количество «НБ», «О» и итоговый % посещаемости (рассчитывается как (П + О) / всего занятий).Визуализация: Цветовая индикация студентов, чей процент посещаемости ниже установленного порога.16.3. Экраны Куратора:Dashboard: Список закрепленных групп с Кнопками и кнопками для перехода к Запросам Б и Просмотр отчётов.Запросы Б: Таблица с очередью студентов, у которых статус Б_pending. Кнопки быстрого действия: «Подтвердить» (зеленая) / «Отклонить» (красная), и кнопкой “Сохранить”Журнал: Таблица студентов с отмеченными статусами посещения и кнопкой для отметки [ А ] и кнопкой “Сохранить”.Экран «Отчеты Куратора»:Фильтры: Выбор конкретной группы из списка закрепленных.Сводная таблица: Посещаемость по всем дисциплинам группы. Отдельный блок «Список Б_pending» (очередь на подтверждение).16.4. СЦЕНАРИИ ВЗАИМОДЕЙСТВИЯ (USER FLOW)Сценарий Преподавателя: Login → Выбор активного занятия → Проставление отметок → Сохранение (пакетное обновление) → Просмотр отчетов → Logout.Сценарий Куратора: Login → Просмотр списка Б_pending → Проверка посещения студентов → Проставление [ А ] → Изменение статуса на Б_confirmed или НБ → Просмотр отчуётов → Logout.Сценарий Учебной части: Login → Поиск конкретного занятия/студента → Корректировка статуса занятия (если нужно) → Формирование аналитического отчета за период → Logout.Сценарий Руководства: Login → Выбор типа отчета (по программам/группам) → Просмотр рейтинга посещаемости → Экспорт данных (если требуется).Сценарий Администратора: Login → Проверка журнала ошибок интеграции → Просмотр Audit Trail для разбора инцидентов.18. СПИСОК API ЭНДПОИНТОВ (ТЕХНИЧЕСКИЙ СТЕК)Общие и Авторизация:POST /auth/login, POST /auth/logout, GET /auth/me.Для Преподавателя:GET /teacher/classes – занятия на сегодня.GET /teacher/class/{id} – список студентов и текущих Attendance.POST /teacher/class/{id}/attendance/update – пакетное сохранение.Для Куратора:GET /curator/groups – список групп.GET /curator/b-requests – список всех Б_pending.POST /curator/b-requests/{id}/approve / reject.Для Учебной части и Руководства:GET /admin/search/students – расширенный поиск.POST /admin/class/{id}/cancel – отмена занятия.POST /admin/semester/dates – установка рамок семестра.GET /reports/attendance/rating – аналитика для руководства.GET /reports/attendance/problem-groups – список групп с низкой посещаемостью.Системные (Администратор):GET /admin/logs/audit – чтение Audit Trail.GET /admin/sync/status – проверка GAUDI/Schedule.
